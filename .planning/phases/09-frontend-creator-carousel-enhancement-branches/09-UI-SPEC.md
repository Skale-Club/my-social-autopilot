> Visual and interaction contract for `post-creator-dialog.tsx` extensions: Carousel and Enhancement branches.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (new-york style) |
| Preset | new-york, baseColor: neutral, cssVariables: true |
| Component library | Radix UI (via shadcn) |

## Spacing Scale

Standard 8-point scale inherited from the existing dialog. All new elements use the same tokens.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon gap inside slide chip label |
| sm | 8px (`gap-2`, `p-2`) | Inline padding for count chips, slide status label gap |
| md | 16px (`gap-4`, `p-4`) | Card grid gap (scenery picker), step section spacing |
| lg | 24px (`gap-6`, `p-6`) | Step body top margin (matches `mt-6` in existing dialog) |
| xl | 32px (`p-8`) | Generating view outer padding (matches existing `p-8`) |
| 2xl | 48px | Not used in new step content |
| 3xl | 64px | Not used in new step content |

**Exceptions:**
- Slide thumbnail placeholders in the progress row: `w-[72px] h-[72px]` per placeholder (fixed square, not on scale) — this is a deliberate compact choice matching the dialog's max-width constraint.
- Touch target minimum: 44px height on all interactive buttons (standard accessibility floor).

## Component Inventory

These are the exact component patterns the executor must implement. Each is derived from existing patterns in the codebase.

### 1. Slide Count Picker ("Slides" step)

**Pattern:** Row of count chips. Each chip is a `<button>` with fixed `w-10 h-10` touch area.

```
[3] [4] [5] [6] [7] [8]
```

- Unselected: `border border-border text-muted-foreground hover:border-violet-400/40 rounded-lg text-sm font-semibold`
- Selected: `border border-violet-400 bg-violet-400/10 text-violet-400 rounded-lg text-sm font-semibold`
- Default selection: 3 slides (lowest valid count)
- Layout: `flex gap-2 flex-wrap justify-start`
- Chip size: `w-10 h-10` (40px × 40px, minimum touch 44px handled by flex padding)

**Rationale:** Chips over a slider — discrete values 3–8 (only 6 options) are better served by direct tap than a slider range. Matches the pill/chip treatment used in the resolution and duration pickers in the Format step.

### 2. Upload Photo Step ("Upload Photo" step — Enhancement branch)

**Pattern:** Large drop zone card, identical to the Phase 8 `SceneriesCard` upload zone, adapted for in-dialog use.

```
┌──────────────────────────────────────────────┐
│                                              │  ← aspect-video ratio
│    [Upload icon]                             │
│    Click to upload  /  Drag & drop           │
│    JPEG, PNG, WEBP · max 5 MB               │
│    Any aspect ratio accepted                 │
└──────────────────────────────────────────────┘
```

- Empty state: `border-2 border-dashed rounded-xl bg-muted/30 aspect-video flex flex-col items-center justify-center gap-2 text-sm text-muted-foreground cursor-pointer hover:bg-muted/40 transition-colors`
- Active drag state: `border-violet-400 bg-violet-400/10`
- Filled state (file selected): shows `URL.createObjectURL()` preview image at `object-cover` within the same `aspect-video` container, plus a remove button (`w-7 h-7 rounded-full bg-black/60 text-white`) at `top-2 right-2` — **the remove button MUST carry `aria-label="Remove photo"`** (icon-only, no visible text label)
- File name shown below the zone: `text-xs text-muted-foreground truncate mt-1`
- Accepts: `image/jpeg,image/png,image/webp`
- Client rejection path: destructive toast with title "File too large" or "Invalid file type" (matches existing dialog validation copy verbatim — see Copywriting Contract)

**Source:** `sceneries-card.tsx` lines 383–407 (upload zone pattern).

### 3. Scenery Picker ("Scenery Picker" step — Enhancement branch)

**Pattern:** Responsive card grid, mirroring the Phase 8 admin SceneriesCard display grid.

```
grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-3 gap-4
```

Each card:
- Container: `rounded-xl border-2 overflow-hidden flex flex-col transition-all cursor-pointer`
- Unselected: `border-border hover:border-violet-400/40`
- Selected: `border-violet-400 bg-violet-400/10`
- Thumbnail: `aspect-video bg-muted/40 overflow-hidden relative` containing `<img object-cover>` or `<ImageIcon w-8 h-8 text-muted-foreground/50 aria-hidden="true">` placeholder when `preview_image_url` is null — the `<ImageIcon>` is decorative and must carry `aria-hidden="true"` (the card's accessible name comes from the card label text)
- Card body: `p-3 flex flex-col gap-1`
  - Label: `text-sm font-semibold truncate`
  - Prompt snippet: `text-[10px] text-muted-foreground line-clamp-1` (first line only, truncated)
- Selection interaction: clicking anywhere on the card sets `sceneryId` — no separate radio/checkbox
- Max scrollable height inside dialog step: `max-h-[340px] overflow-y-auto` when card count exceeds visible area

**Empty state (zero active sceneries):** Never reached in this flow — Enhancement is hidden in Content Type step when active sceneries are zero (D-15). This guard means the Scenery Picker step always renders with at least one card.

**Source:** `sceneries-card.tsx` lines 264–334 (card grid display), `post-creator-dialog.tsx` selection class pattern.

### 4. Content Type Step (extended — 4 types)

**Pattern:** `grid grid-cols-2 gap-4` when 2–3 types enabled. `grid grid-cols-2 sm:grid-cols-4 gap-4` when all 4 types enabled. Hidden entirely when only 1 type enabled (pre-set, no picker shown).

Each content type card:
- Same `p-6 rounded-xl border-2` pattern as existing Image/Video cards
- Icon container: `w-16 h-16 rounded-full flex items-center justify-center`
- Selected icon container: `bg-violet-400/20`, icon: `text-violet-400`
- Unselected icon container: `bg-muted`, icon: `text-muted-foreground`
- New icons:
  - Carousel: `LayoutPanelTop` from lucide-react (grid/panel metaphor)
  - Enhancement: `Sparkles` from lucide-react (already imported; alternately `Wand2` — executor may choose either, must be consistent)
- All icons inside content type cards are decorative (card label text provides the accessible name); add `aria-hidden="true"` to each icon element.
- Enhancement card is hidden (not rendered) when `activeSceneries.length === 0` per D-15. When hidden, a note below the grid reads: `"Photo enhancement is currently unavailable."` — `text-xs text-muted-foreground text-center mt-2`

### 5. Carousel Generating State (adapted `"generating"` view)

**Layout:** Same centered flex column as existing generating view, with one addition — the slide thumbnail row above the loader.

```
┌─────────────────────────────────────────────────────┐
│   Generating slide 2 of 5…          [status text]  │
│                                                     │
│   [□] [◉] [□] [□] [□]              [thumbnail row] │
│   slide 1: done  ↑ slide 2: in progress             │
│                                                     │
│             [GeneratingLoader]                      │
│                                                     │
│   Creating Your Carousel                            │
│   Generating slide 2 of 5…                         │
│                                                     │
│   ████████░░░░░░░░  40%              [Progress]    │
└─────────────────────────────────────────────────────┘
```

**Slide thumbnail row:**
- Container: `flex gap-2 justify-center flex-wrap mb-6`
- Each placeholder: `w-[72px] h-[72px] rounded-lg overflow-hidden relative bg-muted flex items-center justify-center flex-shrink-0`
- **Pending state** (not yet generated): shows `<Loader2 className="w-5 h-5 animate-spin text-muted-foreground" aria-hidden="true" />` — the spinner is decorative; status is communicated by the status text above the row
- **In-progress state** (current slide being generated): shows `<Loader2 className="w-5 h-5 animate-spin text-violet-400" aria-hidden="true" />` + pulsing ring: `ring-2 ring-violet-400/60`
- **Success state** (slide_complete event received): swaps to `<img src={slideImageUrl} className="w-full h-full object-cover" alt={`Slide ${slideIndex}`} />` with a fade-in transition (`transition-opacity duration-300`)
- **Failed state** (slide_failed event received): shows `<AlertTriangle className="w-5 h-5 text-destructive" aria-hidden="true" />` centered, red tint background `bg-destructive/10` — the slide placeholder wrapping element carries `aria-label={`Slide ${slideIndex} failed`}` so screen readers can report the failure
- Slide index: `absolute bottom-1 left-1 text-[9px] text-white bg-black/60 px-1 rounded` showing slide number (e.g. "1") — decorative duplicate of the `aria-label`; add `aria-hidden="true"` to this element

**Status text above row:** `text-sm text-muted-foreground text-center mb-4` — "Generating slide {n} of {total}…" — updates on each `slide_progress` SSE event.

**Progress bar:** Same `<Progress>` component as existing. Value = `(completedSlides / totalSlides) * 100`. Updates per slide event.

**Heading:** "Creating Your Carousel" (matches existing "Creating Your Post" size: `text-xl font-semibold mb-2`).

### 6. Carousel Result View (`"result"` view mode)

**Layout:** New `viewMode === "result"` state rendered inside the same `AnimatePresence`. Shown after carousel SSE `complete` event fires.

```
┌─────────────────────────────────────────────────────┐
│  Carousel Ready                    [Step indicator] │
│                                                     │
│  [img1] [img2] [img3] [img4] [img5]  [slide grid]  │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │  Caption text here...                         │  │  ← read-only, selectable
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  [Generate Another]           [Save & Close]        │
└─────────────────────────────────────────────────────┘
```

**Heading:** `text-xl font-semibold` — "Carousel Ready"

**Partial-draft notice** (when `status === "draft"`): inline warning block above caption:
- `flex items-center gap-2 p-3 rounded-lg bg-orange-500/10 border border-orange-500/30 text-xs text-orange-400 mb-3`
- Icon: `<AlertTriangle className="w-4 h-4 flex-shrink-0" aria-hidden="true" />`
- Text: "Only {n} of {requested} slides were generated. Your post was saved as a draft."

**Slide grid:** Same layout as thumbnail row from generating state — all successful slides shown as real images. No spinners. No failed-slide placeholders. `flex gap-2 flex-wrap justify-center mb-6`

**Caption block:**
- `w-full rounded-lg border border-border bg-muted/30 p-3 text-sm text-foreground select-text cursor-text max-h-[120px] overflow-y-auto`
- Label above: `text-xs font-semibold text-muted-foreground uppercase tracking-wide mb-2` — "Caption"
- Read-only. User can select and copy text.

**Action row:** `flex items-center justify-between gap-3 mt-6`
- Left: `<Button variant="ghost">Generate Another</Button>` — resets to Content Type step (full state reset per D-22)
- Right: `<Button>Save & Close</Button>` — calls `closeCreator()` (markCreated already called on SSE complete)

### 7. Carousel Format Step (locked aspect ratios)

Uses the same Format step UI as Image/Video. Only the available options differ — only `1:1` and `4:5` cards are rendered. No 9:16 or 16:9 options.

**Note banner** below the format cards:
- `flex items-start gap-2 p-3 rounded-lg bg-violet-400/5 border border-violet-400/20 mt-3`
- Icon: `<Info className="w-4 h-4 text-violet-400 mt-0.5 flex-shrink-0" aria-hidden="true" />`
- Text: `text-xs text-muted-foreground` — "All slides in this carousel share the same format."

## Interaction States

### Validation — Generate Button Disabled When

**Carousel branch:**
- `referenceText.trim() === ""` AND `referenceImages.length === 0` (at least one required)
- `slideCount < 3` OR `slideCount > 8`
- `aspectRatio` not in `["1:1", "4:5"]`

**Enhancement branch:**
- `enhancementFile === null` (no photo uploaded)
- `sceneryId === null` (no scenery selected)

The "Next" button between steps is never disabled — validation only blocks the final Generate/Enhance button on the last step.

### Transitions

All step transitions use the existing `<AnimatePresence mode="wait">` with `<motion.div key={step-{step}}>` + `initial={{ opacity: 0, y: 12 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -12 }}`. Do not change the animation config.

The transition from `"generating"` → `"result"` uses `initial={{ opacity: 0, scale: 0.96 }} animate={{ opacity: 1, scale: 1 }}` (matches existing `"generating"` entry animation).

### Reset on Content Type Change (D-22)

When the user navigates back to the Content Type step and selects a different type: call a `resetBranchState()` helper that zeroes out:
- `slideCount` (back to 3)
- `enhancementFile` (null)
- `sceneryId` (null)
- `postMood` (back to `defaultPostMood`)
- `aspectRatio` (back to `"1:1"`)
- `referenceText` (empty)
- `referenceImages` (empty array)

The step counter resets to 1 (the first step after Content Type).
